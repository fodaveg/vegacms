Notas de dirección — C · «Cabina»

  • Idea: instrumento de precisión. Base 13px, filas de 36px, retícula visible, meta y estados en monoespaciada (published/draft tal cual los valores canónicos de P2 §4.5), atajos de teclado a la vista (/ buscar, N nueva, ⌘S guardar).
  • Topbar propia con búsqueda global y estado de conexión con PB — la dirección que más «panel de control» se siente, la más cercana al usuario dev que instala Vega.
  • Editor con label a la izquierda (168px) y secciones como fichas: máxima densidad de campos por pantalla sin perder escaneabilidad; los fieldGroups de P2 son literalmente las fichas.
  • Pantalla 3 = el editor del manifiesto de P2 §6.4: JSON crudo + dry-run con warnings reales del contrato (icon-unknown, list-field-unknown) con su path, y Guardar deshabilitado hasta validar. En esta dirección esa pantalla no desentona: es una más.
  • Estados mostrados: filtros por estado, fila seleccionada, error de slug, readonly (fecha fijada al publicar + tira de estados), «Ver en el sitio» deshabilitado con motivo, badge de avisos del manifiesto, título kilométrico truncado.
  • Riesgo asumido: es la menos «la edición es el producto» de las tres — eficiente para gestionar, menos agradable para escribir largo. Casaría bien como MODO (densidad compacta) sobre la dirección A o B, más que como identidad única.

La web tenía diecinueve paletas de color y un problema precioso: nadie iba a Ajustes a cambiarlas. La solución no fue un menú mejor — fue un dado.

Tirar el dado

Cada visita puede pulsar el 🎲 de la esquina y la web se repinta con otra paleta al azar: papel, tinta y acento cambian de golpe, y de repente el sitio parece otro sin dejar de ser el mismo. Es la demo del sistema de temas contada sin una sola palabra.

Un sistema de diseño se explica mejor jugando con él que documentándolo.

Debajo hay poca magia: cada tema son siete valores hex y un tinte que se mezcla con los neutros. Lo interesante es que el mismo motor sirve para una web pública, un planificador y — pronto — un panel de administración.

Notas de dirección — B · «Cuaderno»

  • Idea: el heredero natural de Lumbre: papel tintado que se nota (bg al 10–12% de tinte, más que el 8% de Lumbre), radios generosos, pastillas, sombras cálidas. El admin como cuaderno agradable, no como herramienta fría.
  • Sidebar con iconos de trazo (set propio de Vega, los ids de icon del manifiesto P2) y activo en pastilla accent-soft; contadores por colección.
  • Listado en tarjeta con filas de dos líneas (título + slug), badges pill (publicada = success-soft, borrador = neutro, «archivada» = badge neutro info según P2 §4.5) y hover accent-soft.
  • Editor a dos columnas: hoja de escritura + panel lateral pegajoso de metadatos (extracto, imagen con preview, chips, relación, fecha con error de validación, switch, JSON plegado). La barra de guardado pegajosa inferior encarna «cambios sin guardar» y dice QUÉ campo cambió.
  • Estados mostrados: búsqueda sin resultados con salida («crea una entrada con ese título»), error de datetime, badge de avisos del manifiesto, toast flotante de guardado, título kilométrico en fila de altura fija.
  • Riesgo asumido: es la dirección más «app» de las tres; si Vega debe sentirse neutro para sitios corporativos, A o C viajan mejor.
